# CHANGES

## Purpose
Store change briefs for incremental updates to existing behavior.

## File naming
Use one file per change brief:
`YYYY-MM-DD-short-kebab-name.md`

Rules:
- Use lowercase letters, numbers, and hyphens only.
- Keep names specific and concise.
- Do not reuse a filename for different changes.

## Brief format
Each brief must include, in order:
1. Title
2. Status (`planned`, `in-progress`, `done`, `rolled-back`)
3. Reason
4. Change summary
5. Impact
6. Validation
7. Rollback plan
8. Follow-ups

Keep entries brief, concrete, and verifiable.
