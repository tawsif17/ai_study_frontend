# FEATURES

## Purpose
Store feature briefs for proposed or delivered features.

## File naming
Use one file per feature brief:
`YYYY-MM-DD-short-kebab-name.md`

Rules:
- Use lowercase letters, numbers, and hyphens only.
- Keep names specific and concise.
- Do not rename files after review starts.

## Brief format
Each brief must include, in order:
1. Title
2. Status (`proposed`, `approved`, `in-progress`, `done`, `rejected`)
3. Problem
4. Scope
5. Out of scope
6. Acceptance criteria
7. Risks
8. Open questions

Keep content factual and testable.
